### 1.进程优先级

```

1.进程保活的使用
	IM服务，定位服务，推送服务。都需要进程保活。
	锁屏服务。
	
2.进程保活和内存消耗取平衡。
	1.被kill，
	2.人为的
	
3.进程优先级
	1.前台进程
	2.可见进程 -- 内存不够了
	3.服务进程 -- 播放音乐，网络操作，数据计算
	4.后台进程 -- 点击返回键，home到桌面。lru的算法
	5.空进程 -- 缓存。缩短用户的启动时间
	
```

### 2.android进程的回收策略

```
1.low memory kill:
	Low memory killer:通过一些比较复杂的评分机制，对进程进行打分，然后将分数高的进程判定为bad进程，杀死并释放内存。
	
2.low memory kill:
	OOM_ ODJ:判别进程的优先级。值越小优先级越高，越不会被杀死
	
```
